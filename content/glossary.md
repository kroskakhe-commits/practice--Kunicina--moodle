1. Алгоритм — точная последовательность действий (инструкций), выполнение которых приводит к решению поставленной задачи.
2. Дискретность — свойство алгоритма, означающее, что он состоит из отдельных шагов, выполняемых последовательно.
3. Детерминированность — свойство алгоритма, означающее, что каждая команда понятна исполнителю однозначно.
4. Результативность — свойство алгоритма, означающее, что он обязательно приводит к результату за конечное число шагов.
5. Массовость — свойство алгоритма, означающее, что его можно применять для решения целого класса однотипных задач.
6. Исполнитель — объект (человек, робот, компьютер), который выполняет команды алгоритма.
7. Линейный алгоритм — алгоритм, в котором все действия выполняются строго друг за другом, по порядку.
8. Блок-схема — графический способ записи алгоритма с помощью геометрических фигур, соединённых стрелками.
9. Псевдокод — способ записи алгоритма, сочетающий элементы естественного языка и конструкции языков программирования.
10. Разветвляющийся алгоритм — алгоритм, в котором последовательность действий зависит от выполнения условия.
11. Ветвление — конструкция алгоритма, обеспечивающая выбор одного из вариантов действий в зависимости от условия.
12. Условие — логическое выражение, которое может быть истинным или ложным.
13. Полная форма ветвления — конструкция «ЕСЛИ-ТО-ИНАЧЕ», где выполняются разные действия в зависимости от условия.
14. Неполная форма ветвления — конструкция «ЕСЛИ-ТО», где действие выполняется только если условие истинно.
15. Циклический алгоритм — алгоритм, в котором определённые действия повторяются несколько раз.
16. Цикл — конструкция алгоритма, обеспечивающая многократное выполнение одной и той же последовательности действий.
17. Тело цикла — последовательность действий, которая повторяется в цикле.
18. Цикл с предусловием — цикл, который выполняется, пока условие истинно; проверка условия происходит до выполнения тела цикла.
19. Цикл со счётчиком — цикл, который выполняется заданное количество раз с использованием переменной-счётчика.
20. Бесконечный цикл — цикл, условие выхода из которого никогда не становится ложным.
21. Массив — структура данных, которая хранит набор элементов одного типа, идентифицируемых по индексу.
22. Структура данных — способ организации и хранения данных в компьютере для эффективного доступа и обработки.
23. Элемент массива — одна ячейка массива, хранящая конкретное значение.
24. Индекс массива — порядковый номер элемента в массиве (в программировании отсчёт начинается с 0).
25. Сортировка массива — процесс упорядочивания элементов массива по возрастанию или убыванию.
